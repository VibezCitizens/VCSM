# BEHAVIOR — bottom-bar

**Module:** `features/shell/modules/bottom-bar`
**Status:** ACTIVE
**Last Updated:** 2026-06-06

---

## §1 What This Module Does

The bottom-bar module owns the persistent navigation rail and the floating leads chip rendered at the app root. It is always mounted in `RootLayout` and hidden via CSS `display:none` when not applicable — this preserves React Query subscriptions and realtime listeners across route changes.

---

## §2 Components

| Component | Mount Pattern | Purpose |
|---|---|---|
| `BottomNavBar` | Always-mounted, CSS-hidden | Primary app navigation rail |
| `VportLeadsChip` | Always-mounted, CSS-hidden | Floating badge for Vport owners with new leads |

---

## §3 Hooks

| Hook | Purpose |
|---|---|
| `useBottomNavVisibility` | Computes `hideBottomNav` from current pathname |
| `useVportLeadsCount` | Polls and subscribes to new leads count for a Vport actor |

---

## §4 Visibility Rules

Bottom nav is hidden when the current route matches any of:
- `/chat/*` sub-screens (not the inbox root `/chat`)
- Auth routes: `/login`, `/register`, `/reset`, `/forgot-password`, `/onboarding`
- Learning routes: `/learning/*`
- Dev performance routes: `/dev/performance*`

---

## §5 Security Rules

- `actorId` must NEVER appear as a raw UUID in any link or navigation path generated by these components. Profile navigation uses `useActorCanonicalSlug(actorId)` from `profiles.adapter` with a `/profile/self` fallback on cold cache. `VportLeadsChip` navigation uses `useActorCanonicalSlug` to build a `/vport/{canonicalSlug}/dashboard/leads` path; the redirect resolves canonical slugs to UUID before reaching `OwnerOnlyDashboardGuard`.
- `VportLeadsChip` navigation is slug-safe — ELEK-002 / BW-BN-002 CLOSED (2026-06-06, TICKET-BOTTOMNAV-SLUG-LEADS-ROUTE-001).
- `BottomNavBar` dispatches `noti:refresh` only when the user navigates to `/notifications` or `/chat` — never on arbitrary routes.
- `window.OneSignal` reference is frozen on first acquisition via `_frozenSdk` module-level cache — ELEK-001 / BW-BN-001 CLOSED (2026-06-06, Batch 1 M-1).

---

## §6 Dependency Rules

This module MAY consume:
- `features/identity/adapters/identity.adapter` (correct path)
- `features/profiles/adapters/profiles.adapter` (correct path — NOT the controller directly)
- `bootstrap/*` (hydration + selectors)
- `shared/hooks/*`

This module MUST NOT:
- Import from any other feature's internal files directly (only via adapters)
- Export anything other than hooks, components, and constants

---

## §9 Must Never Happen

- MUST NEVER render `BottomNavBar` outside the `RootLayout` tree
- MUST NEVER remove `BottomNavBar` from the DOM on route change (CSS hide only)
- MUST NEVER dispatch `noti:refresh` without the `/notifications` or `/chat` pathname guard
- MUST NEVER expose raw `actorId` UUID in any user-visible URL or navigation path
- MUST NEVER allow `window.OneSignal` to be replaced mid-session without re-binding the auth listener
