Automate Traffic static-export rebuilds on provider-data changes (Supabase webhook → deploy hook / CI)

**Context:**
Traffic is a static-export site (`output: "export"` in `next.config.mjs`). All provider, city, category, and index pages are generated at build time from a build-time snapshot of `vport.public_traze_provider_index_v` (loaded once via top-level `await` in `src/data/connectors/unifiedDataset.js`). After deployment there is no server-side rendering and no re-query of the database, so the live site keeps serving the snapshot from the last build. This is the proven root cause of TICKET-TRAZE-CITY-FILTER-001: provider `gas-1ca2` (Laredo) existed in the DB but did not appear on production until a fresh `npm run build` regenerated the snapshot. Manual build-and-upload on every provider change is error-prone and does not scale.

**Prompt Origin:**

Traffic is currently configured as a static-export site, which means all provider pages, city pages, categories, and indexes are generated at build time and then served as static files. When you add or update a provider like gas-1ca2, the database updates immediately, but the live Traffic site does not automatically re-query the database because there is no server-side rendering happening after deployment. Instead, the site continues serving the snapshot of data that existed when the last build was created. The reason gas-1ca2 appeared after your local build is that npm run build generated a fresh snapshot and included the new provider in the exported routes. To avoid manually building and uploading every time, you can automate the process by connecting Traffic to a deployment pipeline (GitHub Actions, Vercel, Netlify, Cloudflare Pages, etc.) and triggering a rebuild whenever provider data changes. A common pattern is for Supabase to fire a webhook when a VPORT becomes active, listed, claimed, or otherwise indexable; that webhook then triggers a deploy hook or CI workflow, which rebuilds Traffic and publishes the updated static site automatically. This preserves the performance and SEO benefits of static export while ensuring new providers appear on the public site without any manual intervention.
