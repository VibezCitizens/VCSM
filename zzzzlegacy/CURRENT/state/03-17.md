# Planning — May / 03 / Sequence 17

## Task

Fix Business Card Preview from Settings — no close/back navigation

## Root Cause

File: `apps/VCSM/src/features/settings/vports/ui/VportsBusinessCardSection.jsx`

```js
<button onClick={() => window.open(cardUrl, '_blank')} ...>
  Preview
</button>
```

Opens the card in a new tab with no router state. The new tab has no prior history
(length === 1), so browser back does nothing meaningful in PWA/Safari.
No preview signal reaches the card — no back button can conditionally render.

## Approach

Same-tab navigation with React Router state (preferred path from task spec).

Change Preview from `window.open(..., '_blank')` to `navigate('/vport/:slug/card', { state: { fromSettings: true } })`.

- Router state carries the preview intent within the same browsing session
- `navigate(-1)` always works since we arrived from Settings
- No query params needed
- Public visitors who open the card directly have no `location.state` → no back button shown
- Refresh loses state → card reverts to public view (acceptable — it's a full reload)

Why this way:
  - Same-tab navigation preserves router state across the transition without needing query params
  - fromSettings gated strictly — public visitors have no location.state so the button never renders
  - Fixed positioning with safe-area-inset-top makes it mobile/PWA/Safari safe at 44px tap target
  - navigate(-1) always works because the user arrived here from Settings (history.length > 1)
  - Refresh loses state intentionally — full reload = public visitor context
