## Diagnosis

### Issue 1 — Miami hardcoded, real location never used

Root causes:
- `next.config.mjs` → `output: "export"` — static site built once at deploy time. Server-side geo-IP is architecturally impossible.
- `page.jsx:37` → `const DEFAULT_LOCATION_CITY_SLUG = "miami"` — hardcoded build-time constant.
- `page.jsx:191-192` → `defaultLocation` resolves to Miami and is passed as prop to `HomepageSearchPanel`.
- `HomepageSearchPanel.jsx:57` → `useState(defaultLocation?.citySlug ?? "miami")` — Miami is the SSG-hydrated client state.
- `handleUseMyLocation` fires ONLY on button click — no auto-detection on mount.

No browser location permission dialog auto-triggers. User must click. Most users never do.

Fix: On mount, check `navigator.permissions.query({name:'geolocation'})`. If `granted` (user approved before), silently call `getCurrentPosition`, find nearest city via existing haversine logic, call `setLocationSlug`. If `prompt` or `denied`, leave Miami but add a clear "Not in Miami?" hint next to the locate button. Never auto-show the permission dialog on load.

### Issue 2 — Mobile search bar / location row visually cut off

Root causes:
- `.hp-location-row` is `display: flex; gap: 0.48rem` with NO small-screen flex-wrap override.
- `.hp-use-location-btn` has `white-space: nowrap` and does not shrink.
- On screens ≤390px wide, the select + gap + button exceeds the form inner width.
- `hp-search-form` padding `0.72rem` provides only ~11.5px buffer on each side — the button text overflows the right edge.

Header nav cut-off in screenshot is intentional (scrollable at ≤600px). Only the location row is broken.

Fix: Add ≤540px breakpoint — wrap `hp-location-row` vertically: select full-width top row, locate button full-width second row.

### Issue 3 — PWA incomplete (not installable)

Current state:
- `layout.jsx` has `appleWebApp`, `themeColor`, `viewportFit: cover` — correct meta foundation.
- No `public/` directory → no manifest.json, no icons, no service worker.
- App is not installable on homescreen, no offline support, no PWA banner.

Fix:
- Create `public/manifest.json` — TRAZE identity, standalone display, purple theme, icon references.
- Create `public/sw.js` — lightweight vanilla cache SW (no workbox; static export can serve it from `/sw.js`).
- Create `public/icons/icon.svg` — TRAZE SVG icon (production swap to PNG later).
- Update `src/app/layout.jsx` — add `manifest` and `icons.apple`.
- Update `src/shared/components/AppShell.jsx` — register SW on mount via `useEffect`.

Reference: `apps/VCSM/public/manifest.json` + `apps/VCSM/src/sw.js` used as structural reference; adapted for Traffic (simpler, no workbox).
