## Execution Plan

### Scope
Application Scope: VCSM (database layer only)
Task Class: MIXED (DB_REVIEW + IMPLEMENTATION)

### Approach
Migration files will be written to `apps/VCSM/supabase/migrations/` and deployed via the Supabase dashboard SQL editor (Docker unavailable for CLI push).

### Slice 1 ‚Äî Inspect Live Schemas (read-only)
Pull actual table + column definitions for `vport`, `reviews`, `notification`, `moderation`, `identity` from the live DB. Use `supabase inspect db table-stats` and schema-aware queries via the Supabase REST API or CLI.

### Slice 2 ‚Äî `vport` Schema RLS
Write and deploy policies for all vport tables:
- `vport.profiles` ‚Äî owner can manage, public can read active non-deleted
- `vport.profile_categories` ‚Äî owner can manage
- `vport.profile_actor_access` ‚Äî owner can manage
- `vport.portfolio_items`, `vport.portfolio_media`, etc. ‚Äî owner can manage, public read
- `vport.barber_portfolio_details`, `vport.locksmith_portfolio_details` ‚Äî owner can manage

### Slice 3 ‚Äî `reviews` Schema RLS
- `reviews.reviews` ‚Äî author can INSERT/UPDATE own, everyone reads published
- `reviews.review_dimensions` ‚Äî same as reviews
- `reviews.review_revisions` ‚Äî author read/insert own

### Slice 4 ‚Äî `notification` Schema RLS
- `notification.events` ‚Äî INSERT via SECURITY DEFINER only; actor reads own events
- `notification.recipients` ‚Äî actor reads/updates own
- `notification.preferences` ‚Äî actor reads/updates own

### Slice 5 ‚Äî `moderation` Schema RLS
- `moderation.blocks` ‚Äî actor can manage own blocks, read own blocks
- `moderation.reports` ‚Äî actor can insert, service role reads all

### Slice 6 ‚Äî `platform` Schema Hardening
- Lock INSERT/DELETE on all platform tables to `service_role` only (all writes already go through SECURITY DEFINER)
- Add explicit DENY via `WITH CHECK (false)` on INSERT for direct client access

### Slice 7 ‚Äî `identity.actor_directory` RLS
- SELECT: authenticated read (public directory)
- INSERT/UPDATE/DELETE: service_role only (managed by `identity.refresh_actor_directory_row`)

### Slice 8 ‚Äî Migration deploy
Write all policies as a single timestamped migration file and provide the SQL to run in Supabase dashboard.
