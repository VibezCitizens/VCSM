### Why it was done this way
- `resolveFuelKeys` and `buildFuelPriceRows` are pure domain transforms — they belong in the model layer, not inside component useMemo blocks
- `useSubmitBulkFuelPrices` owns the full lifecycle of a bulk submit operation; modal's responsibility is UI state (values, shareToFeed) only
- useMemo wrappers remain in `GasPricesPanel` — React performance concern correctly stays at component layer
- `submitError` kept in `VportGasPricesView` — view-level banner outside `GasPricesPanel`, separate concern
