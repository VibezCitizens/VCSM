## Execution Summary

### Files Created / Modified

**NEW:** `apps/VCSM/src/features/profiles/kinds/vport/dal/exchange/vportExchangeRatePost.read.dal.js`
- `resolveVportExchangeNameDAL(actorId)` — reads `vport.profiles.name` via vportSchema
- `hasRecentExchangeRatePostDAL({ actorId, windowMs })` — 1-hour dedup check against `vc.posts` WHERE actor_id + post_type='exchange_rate_update' + deleted_at IS NULL + created_at >= since

**NEW:** `apps/VCSM/src/features/profiles/kinds/vport/controller/exchange/publishExchangeRateUpdateAsPost.controller.js`
- Calls `resolvePublicRealmIdDAL()` (synchronous constant — void-realm-safe)
- Calls `hasRecentExchangeRatePostDAL` — returns `{ published: false, reason: "dedup_throttle" }` if throttled
- Calls `resolveVportExchangeNameDAL` for display name
- Builds post text: `Exchange rates updated at [name]\n\n${pair} — Buy: X.XXXX · Sell: X.XXXX`
- Calls `createSystemPost` with `post_type: "exchange_rate_update"`

**NEW:** `apps/VCSM/src/features/profiles/kinds/vport/hooks/exchange/usePublishExchangeRatePost.js`
- Thin hook wrapping the controller
- Returns `{ publishExchangeRatePost }`

**MODIFIED:** `apps/VCSM/src/features/profiles/kinds/vport/screens/rates/components/VportRateEditorCard.jsx`
- Added `shareToFeed` / `onToggleShareToFeed` props
- Added "Share to feed" checkbox (inline styles: appearance auto, accentColor sky-400 — matching CSS reset fix pattern)
- Bottom row now flex justify-between: checkbox label left, Publish Rate button right

**MODIFIED:** `apps/VCSM/src/features/dashboard/vport/screens/VportDashboardExchangeScreen.jsx`
- Imported `usePublishExchangeRatePost`
- Added `shareToFeed` state (default false)
- Added `publishExchangeRatePost` from hook
- After successful upsert: if `shareToFeed`, calls `publishExchangeRatePost` in try/catch (non-blocking)
- Passes `shareToFeed` + `onToggleShareToFeed={setShareToFeed}` to `VportRateEditorCard`

### Post Text Format
```
Exchange rates updated at [vport name]

USD/MXN — Buy: 17.5000 · Sell: 17.8000
```

### Architecture Notes
- Feed publish is non-blocking: failure never prevents rate save toast
- Dedup: 1-hour window via `hasRecentExchangeRatePostDAL`
- Void-realm safe: `resolvePublicRealmIdDAL()` returns `PUBLIC_REALM_ID` constant directly
- `upsertVportRateController` has no actor ownership check (pre-existing gap, out of scope)
