## Phase 1 — Audit Results

### Tab System Architecture

**Tab definitions:** `features/profiles/config/profileTabs.config.js`
- Defines all possible tabs via `TAB` catalog
- `TAB_FLAGS` controls feature flags (PORTFOLIO: true currently)
- `makeTabs()` builds frozen tab arrays from key lists
- 6 layout presets: `VPORT_TABS`, `VPORT_SERVICE_TABS`, `VPORT_BARBER_TABS`, `VPORT_FOOD_TABS`, `VPORT_GAS_TABS`, `VPORT_RATES_TABS`

**Tab resolution:** `features/profiles/kinds/vport/model/gas/getVportTabsByType.model.js`
- Maps vport_type → type-specific tabs or group-based tabs
- `TYPE_TABS`: barber → VPORT_BARBER_TABS, locksmith → VPORT_BARBER_TABS, gas station → VPORT_GAS_TABS, exchange → VPORT_RATES_TABS
- `GROUP_TABS`: "Beauty & Wellness" → VPORT_SERVICE_TABS, "Food, Hospitality & Events" → VPORT_FOOD_TABS, Other → VPORT_TABS

**Tab rendering:** `features/profiles/kinds/vport/screens/VportProfileViewScreen.jsx`
- `effectiveTabs` computed from `getVportTabsByType(vportType)` + owner tab injection
- Tab state managed via `useState('vibes')` with fallback logic
- `VportProfileTabs` renders the tab bar
- Tab content rendered via `{tab === 'portfolio' && <VportPortfolioView ... />}` pattern

### Portfolio System

**Portfolio hook:** `features/profiles/kinds/vport/hooks/portfolio/useVportPortfolio.js`
- Fetches from portfolio engine via `ctrlListPortfolio(actorId)`
- Returns: items, loading, hasMore, selectedItem, selectedItemDetail, filterTag, openItem, closeItem

**Portfolio view:** `features/profiles/kinds/vport/screens/portfolio/view/VportPortfolioView.jsx`
- Grid of `PortfolioItemCard` components
- Built-in `PortfolioDetailModal` for item details (hero image, before/after, tags, barber/locksmith details)
- Empty state shown when items.length === 0

### Photo Viewer (Reusable)

**ImageViewerModal:** `features/profiles/screens/views/tabs/photos/components/ImageViewerModal.jsx`
- Full-screen modal with image navigation
- Supports reactions (like, rose), comments, sharing
- Used by `PhotoGrid` in the Photos tab
- Accepts: imagePosts, activeIndex, setActiveIndex, onClose, toggleReaction, etc.

**PhotoGrid:** `features/profiles/screens/views/tabs/photos/components/PhotoGrid.jsx`
- Grid display of photo posts with viewer integration

### Which Layouts Include Portfolio

| Layout | Has Portfolio | VPORT Types |
|---|---|---|
| `VPORT_TABS` (default/Other) | YES (1st tab) | All "Other" types |
| `VPORT_SERVICE_TABS` | YES (1st tab) | Beauty & Wellness |
| `VPORT_BARBER_TABS` | YES (1st tab) | barber, locksmith |
| `VPORT_FOOD_TABS` | YES (2nd tab) | restaurant, catering, etc. |
| `VPORT_GAS_TABS` | NO | gas station |
| `VPORT_RATES_TABS` | NO | exchange |

### Root Cause

Tabs are purely type-based — there's no data-aware check for whether portfolio items actually exist. A restaurant with zero portfolio items still shows the Portfolio tab because `VPORT_FOOD_TABS` includes it statically.

### Reuse Plan

- **Phase 2**: Add a `useHasPortfolio(actorId)` hook (lightweight count check) and filter `effectiveTabs` in `VportProfileViewScreen` to exclude portfolio when empty.
- **Phase 3**: The `PortfolioDetailModal` already exists and works. The issue is it's a basic modal, not using the premium `ImageViewerModal` pattern. We can enhance it to use `ImageViewerModal` for the image viewing portion while keeping the portfolio metadata overlay.
