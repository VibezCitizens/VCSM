# Planning — May / 03 / Sequence 19

## Task

Fix TRAZE — "Show on TRAZE directory" toggle broken + all VPORT profile pages 404

## Root Cause

TRAZE DAL (`vportDataset.read.dal.js`) queried Supabase with TWO filters:
  - `.eq("directory_visible", true)`
  - `.eq("directory_status", "listed")`

VCSM settings toggle ("Show on TRAZE directory") only writes `directory_visible`.
`directory_status` is admin-only — it defaults to `"pending"` and is never set to `"listed"` by
any user-facing action. Result: NO VPORT ever loaded into the TRAZE dataset, regardless of
the toggle state. Every profile page 404d. Directory listings were empty.

Additionally, `isIndexable` in the mapper included `isDirectoryActive` (directory_status === "listed"),
further preventing directory listing eligibility even if the filter were relaxed.

## Execution Summary

What was completed:

1. `vportDataset.read.dal.js`
   - Removed `.eq("directory_visible", true)` and `.eq("directory_status", "listed")` filters
   - All VPORTs from `vport.public_traze_profiles_v` now load into the dataset
   - `isIndexable` in the mapper controls directory listing eligibility (not the DAL filter)

2. `vportToProvider.model.js`
   - Removed `isDirectoryActive` from `isIndexable` computation
   - `isIndexable` is now: `Boolean(row.is_active) && directoryVisible && slug.length > 0`
   - Toggle ON (`directory_visible=true`) → appears in directory listings
   - Toggle OFF → not in directory listings, but still has a profile page

3. `provider.repo.js`
   - Added `getProviderBySlugAny(providerSlug)` — searches all active providers without
     `isIndexable` gate, for profile page lookups
   - Added `listAllActiveProviders()` — returns all active VPORTs

4. `staticParams.repo.js`
   - Added `listAllActiveProviderStaticParams()` — maps all active providers to
     `{ providerSlug }` for static route generation

5. `apps/Traffic/src/app/(seo)/pro/[providerSlug]/page.jsx`
   - `generateStaticParams()` now calls `listAllActiveProviderStaticParams()` first
     (falls back to mock slugs only when Supabase is unavailable at build time)
   - `buildProviderGraph()` now calls `getProviderBySlugAny()` so all active VPORTs
     resolve to a page instead of null → 404

Why this way:
  - `directory_status` was an admin-only field with no user-facing path to set it — it blocked
    a fully self-service feature
  - `directory_visible` (the toggle) is the correct and sufficient gate for directory listings
  - Profile pages and directory listings are now decoupled: ALL active VPORTs get a profile
    page; only toggle-ON VPORTs appear in directory listings
  - No VCSM code changes needed — the toggle was already correct on the VCSM side
