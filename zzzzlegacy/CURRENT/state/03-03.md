Root Cause:
`useIdentity()` returns only `{ actorId, kind, ownerActorId }` — the public identity.
`AccountTab.view.jsx` was reading `identity?.displayName` and `identity?.photoUrl` from this
public object, both always undefined. VPORT mode survived via `activeVport?.name` /
`activeVport?.avatar_url` fallback from the VPORT list. Citizen mode had no fallback.

Fix:
`useIdentityDisplayDeprecated()` reads `displayName`, `username`, `avatar` from the full
`identityDetails` in `IdentityDetailsContext` — the correct source. Added to
`useAccountController.js`, exposed as `displayName`, `avatarUrl`, `handle`.
Both citizen and VPORT paths in the view now use these fields.
