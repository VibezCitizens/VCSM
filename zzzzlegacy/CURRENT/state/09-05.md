- State slugs are lowercase stateCode (e.g., "tx", "ca", "jalisco")
- City slugs are names (e.g., "dallas", "san-francisco")
- No namespace collision: states are codes, cities are names

### Blocking Risks
- `provider.repo.js` is already 337 lines (pre-existing violation). Adding 3 functions risks exceeding 300. Mitigation: keep functions compact (5-10 lines each); if needed split out a `state.repo.js`.
- State slug collision: if a state code ("tx") could match a city slug, `resolveCountryCity` runs first and would win. This is the correct priority ‚Äî city pages take precedence.
- El Salvador/Guatemala states are called "departments" ‚Äî the data uses `primaryRegionCode` for all; the template should label generically ("region") not "state".
