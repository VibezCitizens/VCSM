## Execution Summary

### Root Cause (confirmed during execution)

There is no splash/background image on the `/onboarding` route. The background is a
pure inline CSS radial-gradient — renders instantly, zero network dependency.

The perceived "slow" load was the `if (loading) return (...)` early return in
`Onboarding.jsx`. While `getOnboardingBootstrapController()` was in flight, the screen
showed a plain black background with `"Preparing onboarding..."`. The gradient card
only appeared after the async call completed, creating a jarring visual delay.

### Fix Applied
- Removed the `if (loading) return (...)` loading gate entirely
- Gradient background + card now render immediately on mount
- Form section wrapped in `transition-opacity duration-200` — fades from 50% opacity
  while loading, to 100% when ready (`pointerEvents: none` during load)
- All inputs get `disabled={loading}` + `cursor-not-allowed`
- Button label: "Loading..." while bootstrap resolves, "Saving..." while submitting
- Submit disabled on `!isValid || loading || saving` (combined as `busy`)
- No layout shift, no new dependencies, no auth logic changed
