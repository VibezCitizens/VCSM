## Common failure patterns across the ranked list

| Pattern | Where it appears | Why it matters |
| --- | --- | --- |
| Remote upload before DB persistence | posts, profiles, VPORT create, menu items, design studio, Wanders builder, chat image attach | Creates orphaned Cloudflare objects and inconsistent UI references |
| Client-owned multi-write orchestration | onboarding, follow accept, block cleanup, fuel review, booking duration, moderation resolution | Leaves the system vulnerable to partial writes and retry duplication |
| Hybrid write authority | chat, block/unblock, notifications creation, VPORT create + services | Makes bug ownership unclear and migration harder |
| Best-effort side effects after the authoritative write | mentions, inbox fan-out, report events, Wanders mailbox/event writes | Users see “successful” actions with missing derived state |

## Stabilization priority

1. Collapse the multi-step client writes into RPC-backed boundaries for onboarding, follow accept, block cleanup, booking duration, and moderation resolution.
2. Add explicit upload finalization or compensating cleanup for every Cloudflare-first mutation.
3. Finish decommissioning legacy `vc.*` chat writes so message send/read/spam all share one engine authority.
4. Remove best-effort mutation tails where the derived state is user-visible, or surface them as repairable jobs instead of silent warnings.
