## Execution Plan

Fix: add `invalidateMenuCache(actorId)` before `refresh()` in `categoriesMut.onSuccess`. `invalidateMenuCache` was already imported in the prior task (19-11).

## Execution Summary

### What was completed

Added `invalidateMenuCache(actorId)` before `refresh()` in `categoriesMut.onSuccess` inside `VportActorMenuManagePanel.jsx`.

This ensures every categories mutation (create, edit, **delete**) busts the 60s TTL cache before the background sync, so `refresh()` always fetches live data from the DB.

The create/edit path retains the `patchMenu` optimistic update for immediate visual feedback. The delete path now works correctly: cache is cleared → `refresh()` fetches the category list without the deleted entry → category disappears from the UI without a manual reload.
