# 12-25 — Fix Vox Chat Badge Unread Source

## Task
Fix the Vox (chat) badge so it counts unread chat messages instead of notification unread counts.

## Root Cause
`inboxUnread.controller.js` delegated to `notificationsCount.controller.js` which calls the notification engine's `countUnread()`. The Vox badge was showing notification unreads, not chat unreads.

## Fix Applied

### File Modified
`apps/VCSM/src/features/notifications/inbox/controller/inboxUnread.controller.js`

**Before:** Imported `getUnreadNotificationCount` from `notificationsCount.controller` (notification engine).

**After:** Queries `chat.inbox_entries` directly via Supabase, sums `unread_count` for all non-archived entries belonging to the actor.

### What Was Preserved (unchanged)
- `useUnreadBadge.js` — hook logic, cache TTL (10s), poll interval, realtime debounce
- `badgeSubscriptions.js` — `subscribeInboxBadge` already watched `chat.inbox_entries` (was correct)
- `BottomNavBar.jsx` — consumption unchanged
- `useNotiCount.js` — notification badge remains on notification engine (correct)

### Data Flow (After Fix)
```
BottomNavBar → useUnreadBadge → loadUnread → getInboxUnreadBadgeCount
  → supabase.schema('chat').from('inbox_entries').select('unread_count')
  → sum all rows where actor_id matches, archived=false, archived_until_new=false
```

### Realtime Path (unchanged)
```
chat.inbox_entries INSERT/UPDATE/DELETE → subscribeInboxBadge → debounce 2s → loadUnread(force=true)
```

## NOTE OF COMPLETITION
Task 12-25 complete. Single file changed. Chat badge now reads from chat.inbox_entries instead of notification engine.
