## Bug Summary

**Symptom:** Menu QR code, Open Menu button, and Copy Link all generate `${origin}/m/${actorId}` where `actorId` is a raw UUID (e.g. `5c2c5402-ba0c-4d84-b628-e52dc3924f9d`). The short `/m/:actorId` redirect then sends users to `/actor/${actorId}/menu` ‚Äî still UUID-based. No human-readable URL exists in the menu share flow.

**Root cause:** `VportActorMenuQrView.jsx` hardcodes `menuUrl = ${origin}/m/${actorId}`. The redirect at `/m/:actorId` bounces to `/actor/${actorId}/menu` (still UUID). No slug-based route exists for public menus.

---

## Investigation Findings

### Current flow:
```
Owner dashboard ‚Üí /actor/{actorId}/menu/qr ‚Üí VportActorMenuQrView
  menuUrl = ${origin}/m/${actorId}   ‚Üê raw UUID

Customer scans QR ‚Üí /m/{actorId} ‚Üí VportPublicMenuRedirectScreen
  ‚Üí navigate(/actor/${actorId}/menu)  ‚Üê still UUID

Old links: /actor/{actorId}/menu ‚Üí VportPublicMenuScreen (actorId-based, no redirect)
```

### Files involved in URL generation:
| File | Current URL |
|------|-------------|
| `features/dashboard/qrcode/menu/VportActorMenuQrView.jsx` | `${origin}/m/${actorId}` |
| `features/dashboard/flyerBuilder/screens/VportActorMenuFlyerView.jsx` | `${origin}/m/${actorId}` |
| `features/public/screens/VportPublicMenuRedirectScreen.jsx` | redirects ‚Üí `/actor/${actorId}/menu` |
| `features/public/vportMenu/screen/VportPublicMenuScreen.jsx` | actorId-based, no redirect |

### Slug source available:
- `buildActorCanonicalSlugController(actorId)` ‚Üí `canonicalSlug` (e.g. `yankee-gasolina-laredo-tx`)
- `useActorCanonicalSlug(actorId)` hook wraps it, cached 10min TTL
- This is the same slug already used for `/profile/:slug` URLs

### New canonical menu URL shape:
```
/profile/:slug/menu
```
Consistent with the existing profile URL pattern (`/profile/:slug`).

## Execution Plan

### Slice 1 ‚Äî Add new slug-based public route (no redirect yet)

**1a. New screen:** `features/public/vportMenu/screen/VportPublicMenuBySlugScreen.jsx`
- Reads `:slug` from `useParams()`
- Uses `useResolveActorBySlug(slug)` ‚Üí actorId
- Renders `<VportPublicMenuView actorId={resolvedActorId} />`
- Shows loading state while resolving
- Shows not-found if slug doesn't resolve

**1b. Route registration:** `app/routes/public/vportMenu.routes.jsx`
- Add `{ path: "/profile/:slug/menu", element: <VportMenuBySlugScreen /> }`

**1c. Lazy import:** `app/routes/index.jsx`
- Add `VportMenuBySlugScreen` lazy import

---

### Slice 2 ‚Äî Fix URL generation (QR + flyer)

**2a. `VportActorMenuQrView.jsx`:**
- Import `useActorCanonicalSlug`
- `const { canonicalSlug } = useActorCanonicalSlug(actorId)`
- `menuUrl = canonicalSlug ? ${origin}/profile/${canonicalSlug}/menu : ${origin}/m/${actorId}`
- Fallback to `/m/${actorId}` if slug not yet resolved (loading) or unavailable

**2b. `VportActorMenuFlyerView.jsx`:**
- Same pattern ‚Äî import `useActorCanonicalSlug`
- Change `menuUrl` from `/m/${actorId}` to `/profile/${canonicalSlug}/menu`
- Fallback to `/m/${actorId}`
- Update `buildShortUrl` to use the new canonical URL (currently shows `vibezcitizens.com/menu/${slug}` ‚Äî keep as display variant only)

---

### Slice 3 ‚Äî Fix redirects (backward compat for old URLs)

**3a. `VportPublicMenuRedirectScreen.jsx`** (`/m/:actorId`):
- Currently: `navigate(/actor/${actorId}/menu)`
- New: `buildActorCanonicalSlugController(actorId)` ‚Üí redirect to `/profile/${canonicalSlug}/menu`
- Fallback: `/actor/${actorId}/menu` if slug resolution fails

**3b. `VportPublicMenuScreen.jsx`** (`/actor/:actorId/menu`):
- Add `useActorCanonicalSlug(actorId)`
- When `canonicalSlug` resolves: `<Navigate to={/profile/${canonicalSlug}/menu} replace />`
- Render menu content while slug is loading (no flash)
- After slug loads, redirect ‚Äî handles old bookmarked actorId-based links
