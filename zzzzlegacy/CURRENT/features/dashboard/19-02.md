## Execution Summary — Content Page Templates

**Task:** VPORT owners can now choose a template (or start from blank) when creating a new content page.

**What was built:**

- **Template definition layer** (`model/contentPageTemplates.js`) — business-type-aware templates for barber, restaurant, gas, exchange, locksmith, plus universal templates (About Us, FAQ, Tips, Why Choose Us). `getTemplatesForVportType(vportType)` returns type-specific first, then universal.
- **Template picker UI** (`components/VportContentTemplatePicker.jsx`) — shown before the form in create mode. Card list with label, category badge, and summary preview. "Start from blank" dashed-border option at bottom.
- **Form prefill** (`components/VportContentPageForm.jsx`) — added `initialValues` prop. Template selection passes `{ title, summary, body, category }` as initial values. All fields remain editable. Fixed pre-existing bug: edit mode now initializes `summary` from `page.excerpt` (correct model field) instead of `page.summary` (undefined).
- **Template step orchestration** (`VportContentManageView.jsx`) — uses a `TEMPLATE_PENDING` Symbol sentinel to distinguish "picker showing" from "null = blank chosen" vs "object = template chosen". Fixed pre-existing `summary`→`excerpt` mapping in `handleSubmit` so excerpt actually saves to DB. Accepts `vportType` prop.
- **vportType wiring** (`VportContentView.jsx`) — extracts `profile.vportType ?? profile.vport_type` and passes to `VportContentManageView`.

**Files changed:**
- `apps/VCSM/src/features/profiles/kinds/vport/screens/content/model/contentPageTemplates.js` (new)
- `apps/VCSM/src/features/profiles/kinds/vport/screens/content/components/VportContentTemplatePicker.jsx` (new)
- `apps/VCSM/src/features/profiles/kinds/vport/screens/content/components/VportContentPageForm.jsx`
- `apps/VCSM/src/features/profiles/kinds/vport/screens/content/VportContentManageView.jsx`
- `apps/VCSM/src/features/profiles/kinds/vport/screens/content/VportContentView.jsx`

**Pre-existing bugs fixed as part of this task:**
- Form initialized `summary` from `page.summary` (undefined) instead of `page.excerpt` — edit mode never pre-populated the summary field
- `handleSubmit` spread `summary` key but controller expected `excerpt` — excerpt was silently dropped on every create/update

**Regression risk:** Low — no DAL/controller/model changes; template only affects form prefill; existing empty create flow preserved via "Start from blank".
