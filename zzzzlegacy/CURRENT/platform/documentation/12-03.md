# Planning — April / 12 / Sequence 03

## Task

Fix duplicate `chat.inbox_entries` and `vc.notifications` unread-count reads. BottomNavBar mount/unmount cycle causes subscription churn and force-refresh bypassing TTL cache.

### Source

Promoted from `.tp-incoming.md` + Kraven investigation in `logan/marvel/kraven/2026-04-12.badge-duplicate-read-hunt.md`

### Root Cause (confirmed by Kraven)

1. `RootLayout.jsx` conditionally renders `{!hideBottomNav && <BottomNavBar />}` — unmounts on chat sub-screens, learning, auth
2. Each remount creates 6 new realtime subscriptions + fires force-refresh (bypasses cache)
3. Rapid navigation = subscription churn + redundant DB queries

### Fix (3 slices)

1. Keep BottomNavBar always mounted — CSS hide instead of unmount
2. Debounce realtime force-refresh in useNotiCount + useUnreadBadge
3. Validate query reduction
