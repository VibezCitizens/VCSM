  — Name resolver (`resolveVportBarbershopNameDAL`) + dedup checks (`hasRecentBarbershopHoursPostDAL`, `hasRecentBarbershopPortfolioPostDAL`). 1-hour dedup window per `actorId + post_type`.

### Why This Approach

- Used `onSaveSuccess` callback prop on `WeeklyAvailabilityGrid` to avoid moving save logic out of the component — no existing pattern was disrupted.
- `firstMediaUrl` passthrough in `usePortfolioItemSubmit` is backwards compatible — existing callers ignore the second argument.
- All publish calls are non-blocking (try/catch with no rethrow) — primary operations (save hours, create portfolio item) are never affected by feed post failures.
- Realm safety: `resolvePublicRealmIdDAL()` returns a constant — never reads viewer session `realmId`, void-safe by construction.
