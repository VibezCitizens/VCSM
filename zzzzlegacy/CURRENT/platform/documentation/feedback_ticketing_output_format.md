All output for tasks tied to a ticket must reference the ticket ID (e.g. [TICKET-0001]) in the response header or summary.

**Why:** User tracks work through persistent tickets and wants output anchored to the ticket thread for clarity and traceability.

**How to apply:** When working on a ticketed task, open the response with the ticket reference. Summaries, completion reports, and validation results should all carry the ticket ID.
