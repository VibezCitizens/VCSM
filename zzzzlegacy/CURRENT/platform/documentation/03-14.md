## Execution Summary

Root cause confirmed:
  Layer 1: routes/index.jsx:253 — { path: "*", element: <Navigate to="/login" replace /> }
           /sitemap.xml matched no public route → hit * wildcard → redirected to /login.
  Layer 2: No _routes.json existed. Cloudflare Pages could not reliably invoke the Functions
           runtime for /sitemap.xml and /sitemaps/* paths, allowing requests to fall through
           to /* /index.html 200 in _redirects, loading the SPA where Layer 1 fired.

What was completed:
  1. Created apps/VCSM/public/_routes.json with explicit include list covering all
     Function-handled paths (/sitemap.xml, /sitemaps/*, /how-to/*, /lovedrop/*, /m/*,
     /p/*, /profile/*, /vport/*, /wanders/*). This forces Cloudflare Pages to always
     route these paths through the Functions runtime, preventing fallthrough to SPA.

  2. Added three defense-in-depth route entries in routes/index.jsx before the * wildcard:
     { path: "sitemap.xml", element: <Navigate to="/" replace /> }
     { path: "robots.txt",  element: <Navigate to="/" replace /> }
     { path: "sitemaps/*",  element: <Navigate to="/" replace /> }
     If any of these paths somehow reach React Router, they redirect to / not /login,
     preventing the login redirect loop.

  3. Confirmed robots.txt already exists as a static file in public/ — no Function needed.

  4. Confirmed Function code in functions/sitemap.xml.js and functions/sitemaps/*.xml.js
     is correct — right onRequest export, right content-type — no changes needed there.
