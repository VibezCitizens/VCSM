## Execution Summary

**What was completed:**

Fixed a live runtime 404 on the `get_follower_count` RPC.

**Root cause:**
`apps/VCSM/src/features/social/friend/subscribe/dal/subscriberCount.dal.js`
called `supabase.rpc('get_follower_count', ...)` without `.schema('vc')`. Supabase
routes unqualified RPC calls to the `public` schema. The function lives in `vc` schema.
Result: HTTP 404 on every follower count load, count always 0.

**Matching VENOM finding:** V-SUB-008 (LOW severity — now promoted to live bug)

**Fix (1 line):**
Added `.schema('vc')` before `.rpc('get_follower_count', ...)` in `subscriberCount.dal.js`.

**Dual-DAL context:**
- Social layer (`social/friend/subscribe/dal/subscriberCount.dal.js`) — calls `vc.get_follower_count` — now fixed
- VPORT layer (`profiles/kinds/vport/dal/subscribersCount.dal.js`) — calls `vc.count_subscribers` — was already correct

These are two different RPCs serving different purposes. The social DAL drives the
follower count badge on actor profiles. The VPORT DAL drives the subscribers tab count.
