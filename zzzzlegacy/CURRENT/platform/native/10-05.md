## Execution Summary — Phase 3 (Menu Banner Stacking)

### Root Cause
The public menu page had NO banner image rendered at all. `profile.bannerUrl` was available from the data but never used in the layout. The entire header was a flat card with avatar + name inline — no visual layering.

### What Changed

**VportPublicMenuView.jsx** — Added banner + stacking layout:
- Banner image (150px mobile, 200px desktop) rendered above the content card using `profile.bannerUrl`
- Fallback gradient when no banner URL exists
- Content card overlaps banner by `marginTop: -40px` with `zIndex: 2`
- Card uses `--vc-card-bg`, `--vc-border`, `--vc-shadow-elevated` tokens
- Back button redesigned to match dashboard (compact, icon-only on mobile)
- Title weight reduced, muted color
- Search input uses `--vc-border` + `--vc-surface-input` tokens
- Page background updated from cyan/teal to `--vc-gradient-a` / `--vc-gradient-b` theme
- Avatar reduced from 72px to 64px, font weights lightened
- Action buttons (Review, Directions, Call) preserved
