## PURPOSE

This module delivers the full Menu QR pipeline for food and hospitality VPORTs. It has two distinct consumer faces:

**1. Public landing surface** — unauthenticated, slug-routed pages where customers scan a QR code or receive a shared link to view the VPORT's digital menu or reviews. No auth required. No VCSM app shell.

**2. Dashboard owner tools** — authenticated cards surfaced on the VPORT dashboard that allow owners to view, display, and print QR codes and flyers for their menu. Includes Classic Flyer, Poster Flyer, and Printable Sheet variants (table cards, half-page, full poster, sticker sheet).

The module is scoped to the `food` VPORT type group (Food, Hospitality & Events) but the QR primitive and public reviews views are generic and used by all VPORT types that have the `reviews_qr` dashboard card.

### MODULE BOUNDARY WARNING #2
**Location:** `apps/VCSM/src/features/public/vportMenu/view/VportPublicMenuView.jsx:59`
**Module:** vportMenu — back navigation fallback
**Current dependency:** `navigate(\`/profile/${actorId}\`, { replace: true })` — actorId (UUID) used in URL path
**Expected boundary:** Public-facing URLs must never contain raw UUIDs. Back navigation fallback violates the "no raw IDs in public URLs" rule.

## DUPLICATE READ DETECTION

⚠️ **POTENTIAL DUPLICATE: `vport.public_menu_read_model_v` read 3 times per public menu page load**

On a `/profile/:slug/menu` page:
1. `resolveMenuSlugDAL` — queries `public_menu_read_model_v` for slug → actorId
2. `readVportPublicMenuRpcDAL` — queries `public_menu_read_model_v` for full menu data
3. `readVportPublicDetailsRpcDAL` — queries `public_menu_read_model_v` for profile header data

All three hit the same underlying view for the same actor. Only `resolveMenuSlugDAL` is cached (10min TTL). The other two re-fetch on every render cycle.

**Mitigation:** Reads 2 and 3 fetch different column sets. The view does the join once. But three separate round-trips to Supabase occur per page load.

**Suggested improvement:** Combine `readVportPublicMenuRpcDAL` and `readVportPublicDetailsRpcDAL` into a single query that fetches both the profile header fields and the menu rows in one call. Or cache the full view result at controller level.
