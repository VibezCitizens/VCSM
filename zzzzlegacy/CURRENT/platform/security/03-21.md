## Execution Summary

**Root Causes Identified:**

1. **All four files ‚Äî silent failure masking the real error**
   `fetchAllPages` used `if (!res.ok) break` ‚Äî any Supabase error (400, 403, schema not exposed, RLS rejection) silently produced an empty `rows = []` with no trace in Cloudflare logs. This made the "1 error" in GSC completely undiagnosable.

2. **`wanderex-profiles.xml.js` ‚Äî inner join as invisible filter**
   The query select included `profile_public_details!inner(profile_id)`. PostgREST treats this as an inner join ‚Äî any VPORT without a matching `profile_public_details` row is EXCLUDED from results. Since the `profile_id` column selected is just the FK back to `profiles.id`, it provides zero useful data while silently collapsing the result set.
