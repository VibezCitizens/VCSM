# Wolverine Execution Log — Deleted Account Gate

**Date:** 2026-05-10
**Scope:** Feed visibility + profile route gate for deleted/inactive Citizen and Vport accounts

---

## Problem

Posts from soft-deleted Citizen and Vport accounts remained visible in the feed. Navigating to a VPORT profile route for a deleted/inactive account showed a broken or empty screen instead of a clear unavailable state.

**Root cause (feed):**
- `readActorsBundle` was fetching `vport.profiles` via the TRAZE directory view (`vport.public_traze_profiles_v`), which excluded VPORTs not listed in the directory. Active, non-listed VPORTs were missing from `vportMap`, and a missing `vportMap` entry defaulted to visible (fail-open).
- `vc.actors` and `public.profiles` were fetched without `is_deleted = false` filters.
- `feedRowVisibility.model.js` treated `vportEntry === null` as visible.

**Root cause (profile route):**
- `vportPublicDetails.read.dal.js` did not select `is_deleted` from `vport.profiles`, so the model couldn't reflect deletion state.
- `VportProfileViewScreen.jsx` had no gate for `publicDetails === null` (no profile row at all).

---

## Changes Applied

### 1. readActorsBundle — Filter Deleted Actors and Profiles

**File:** `apps/VCSM/src/features/feed/dal/feed.read.actorsBundle.dal.js`

- Added `.eq("is_deleted", false)` to the `vc.actors` query
- Added `.eq("is_deleted", false)` to the `public.profiles` query
- Changed `vport.profiles` query source from `vport.public_traze_profiles_v` to `vport.profiles` directly — selects `actor_id, name, slug, avatar_url, is_active, is_deleted`
- `vportMap` now keyed by `v.actor_id` (correct — matches `vc.actors.id`)

**Effect:** Deleted Citizen actors disappear from `actorMap` → `reason: 'missing_actor'` → `visible: false`. Deleted or inactive VPORTs present in the map but flagged → `reason: 'inactive_vport'` → `visible: false`.

### 2. feedRowVisibility.model.js — Fail-Closed on Missing Vport Profile

**File:** `apps/VCSM/src/features/feed/model/feedRowVisibility.model.js`

Changed VPORT visibility block:

**Before:**
```js
const isActive = vportEntry === null
  || (vportEntry.is_active !== false && vportEntry.is_deleted !== true);
```
(null = visible — fail-open)

**After:**
```js
if (vportEntry === null) {
  return { visible: false, reason: "missing_vport_profile", ... };
}
const isActive = vportEntry.is_active !== false && vportEntry.is_deleted !== true;
```

**New reasons added:** `missing_vport_profile`

### 3. vportPublicDetails.read.dal.js — Added is_deleted to Select

**File:** `apps/VCSM/src/features/profiles/dal/vportPublicDetails.read.dal.js`

Added `is_deleted,` to the explicit column list in the SELECT.

### 4. mapVportPublicDetails.model.js — Map isDeleted

**File:** `apps/VCSM/src/features/profiles/kinds/vport/model/mapVportPublicDetails.model.js`

Added `isDeleted: raw.is_deleted ?? false,` after `isActive: raw.is_active ?? null,`

### 5. NEW: isDeletedProfileActor.model.js

**File:** `apps/VCSM/src/features/profiles/model/isDeletedProfileActor.model.js`

Pure model helper. Returns `true` when an actor's account is no longer publicly available:

```js
export function isDeletedProfileActor({ isDeleted = null, isActive = null } = {}) {
  if (isDeleted === true) return true;
  if (isActive === false) return true;
  return false;
}
```

### 6. NEW: UnavailableProfileGate.jsx

**File:** `apps/VCSM/src/features/profiles/ui/UnavailableProfileGate.jsx`

Rose-toned gate component matching `PrivateProfileGate` structure. Uses `useTranslation()` for i18n. Rendered when a profile route resolves to a deleted/inactive account.

**Adapter:** `apps/VCSM/src/features/profiles/adapters/ui/UnavailableProfileGate.adapter.js`

### 7. VportProfileViewScreen.jsx — Gate Integration

**File:** `apps/VCSM/src/features/profiles/kinds/vport/screens/VportProfileViewScreen.jsx`

```js
const isVportUnavailable =
  !publicDetailsLoading &&
  (
    publicDetails === null ||
    isDeletedProfileActor({ isDeleted: publicDetails?.isDeleted, isActive: publicDetails?.isActive })
  );

if (isVportUnavailable) {
  return (
    <div className="profiles-modern h-full w-full overflow-y-auto touch-pan-y">
      <UnavailableProfileGate />
    </div>
  );
}
```

**Two triggers:** (a) no `vport.profiles` row for the slug → `publicDetails === null`, (b) explicit `isDeleted: true` or `isActive: false`.

### 8. resolveActorSlug.dal.js — Filter Deleted Actors in Slug Resolution

**File:** `apps/VCSM/src/features/profiles/dal/resolveActorSlug.dal.js`

Added `.eq('is_deleted', false)` to the `vc.actors` query in the `public.profiles.username` fallback path. Prevents deleted Citizen actors from resolving via their username slug.

### 9. i18n Keys Added

**Files:**
- `apps/VCSM/src/i18n/en/profile.json`
- `apps/VCSM/src/i18n/es/profile.json`

Keys under `profile.unavailableGate`: `badge`, `title`, `subtitle`.

## Enforcement Flow After Changes

```
User navigates to /v/:slug
  → useVportProfileBySlug(routeSlug)
    → vportPublicDetails.read.dal.js: SELECT id, name, slug, ..., is_active, is_deleted
    → mapVportPublicDetails: { isActive, isDeleted, ... }
  → VportProfileViewScreen: isVportUnavailable check
    → publicDetails === null → UnavailableProfileGate
    → isDeletedProfileActor({ isDeleted, isActive }) → UnavailableProfileGate
    → (otherwise) → normal profile render
```

```
Feed page load
  → readActorsBundle: vc.actors (is_deleted=false), public.profiles (is_deleted=false)
  → vportMap keyed by actor_id from vport.profiles (is_deleted, is_active present)
  → feedRowVisibility:
    → actorMap missing → reason: missing_actor → hidden
    → vportMap missing → reason: missing_vport_profile → hidden
    → vportEntry.is_deleted = true → reason: inactive_vport → hidden
    → vportEntry.is_active = false → reason: inactive_vport → hidden
```

---

## Remaining Gaps

- **Cache lag:** Actor bundle TTL is 30s. A freshly deleted account may appear in the feed for up to 30s before cache expires.
- **DB-level RLS:** `vc.posts` INSERT/SELECT policies do not enforce `is_deleted = false` on the author actor. App-layer filter is the only enforcement.
- **Citizen profile route:** `UnavailableProfileGate` is only wired to the Vport profile screen. Citizen profile gate for deleted accounts is not yet implemented.
