| Tab | filter param | Source | Returns | Notes |
|-----|-------------|--------|---------|-------|

Landing state (no query): NO DB reads from the search stack. Only onboarding cards fetch.

CACHE TOPOLOGY

useSearchScreenController: Manual Map TTL cache (searchResultCache, 45s TTL, max 120 entries)
  - Key format: `${filter}:${query}`
  - In-flight dedup: searchInflight Map (prevents parallel identical requests)
  - Scope: module-level singleton (survives route leave/return)
  - PROBLEM: results from this cache are NEVER RENDERED. The Map caches data into `results` state, but SearchScreen.view.jsx never destructures `results` from useSearchActor().

useSearchTabsActor: NO CACHE
  - Bare useEffect + setState
  - Every search or tab switch = fresh network call
  - Race safety via cancelled ref + requestIdRef monotonic counter ✓
  - No in-flight dedup

useOnboardingCards: NO CACHE
  - Bare useEffect + setState
  - Refetches on every actorId change or component mount

React Query usage in Explore: ZERO
  - No useQuery, no useQueryClient, no queryKeys, no staleTime, no gcTime
  - No keepPreviousData / placeholderData
  - Query keys do NOT include tab/search/actor in any RQ key (because RQ is not used)
  - Cache survives route leave: useSearchScreenController's module-level Map survives; all component state is lost on unmount

### Slice 4 — Root Cause + Fix Plan

ROOT CAUSE OF BLANK EXPLORE BODY
Primary cause: SHOW_EXPLORE_DISCOVERY_BLOCKS = false (ExploreFeed.jsx:4)
This flag is hardcoded false. ExploreFeed always returns null. The only non-search landing content
is OnboardingCardsView, which returns null when:
  - cards.length === 0 (user has no cards, or fetch fails silently)
  - allCompleted === true (user has completed all onboarding steps)
Result: any user with completed onboarding sees a completely blank Explore screen body.

ISSUES RANKED BY SEVERITY

P0 — BLANK LANDING BODY
  ExploreFeed hardcoded off (SHOW_EXPLORE_DISCOVERY_BLOCKS = false).
  No discovery content exists for the non-search state once onboarding is done.
  Impact: every onboarded user sees a blank Explore screen.
  File: apps/VCSM/src/features/explore/ui/ExploreFeed.jsx:4

P1 — DUPLICATE SEARCH FETCH ON EVERY KEYSTROKE
  useSearchScreenController fetches via ctrlSearchResults (results never used).
  ResultList independently fetches via useSearchTabsActor → ctrlSearchTabs.
  Both call identity.search_actor_directory with the same query/filter.
  Impact: 2x RPC calls per search, 2x on tab switch.
  Files:
    hooks/useSearchScreenController.js (fetches, caches, results discarded)
    ui/ResultList.jsx (uses useSearchTabsActor for the actual data)
    controller/searchResults.controller.js (called but rendered nowhere)
    controller/searchTabs.controller.js (called and rendered)

P1 — NO CACHE IN useSearchTabsActor
  Every search / tab switch / Explore re-open triggers a fresh identity RPC call.
  The 45s Map cache in useSearchScreenController is orphaned (its results are never used).
  Impact: cold blank skeleton on every return to Explore.
  File: hooks/useSearchTabsActor.js

P2 — NO React Query — no warm cache, no keepPreviousData
  Tab switch causes full skeleton flash (items=[], loading=true → skeleton renders).
  No staleTime, no background reconcile.
  Impact: UX feels cold; tabs feel disconnected.

P2 — ONBOARDING RETURNS NULL FOR COMPLETED USERS, NO FALLBACK
  OnboardingCardsView.jsx:83-84: if (!cards.length) return null; if (allCompleted) return null
  No fallback content is rendered. Combined with ExploreFeed=null → blank screen.
  File: ui/SearchScreen.view.jsx (no fallback after OnboardingCardsView)

P3 — DEAD CODE: search.usecase.js (broken imports, never imported)
  Imports normalizeResult + dedupeByKindAndId from search.model.js where they don't exist.
  Not a runtime issue (nothing imports it). Safe to delete.
  File: usecases/search.usecase.js

P3 — DEAD CODE: FilterTabs.jsx (never imported)
  Different tab labels from SearchScreen.view.jsx inline FILTERS. Never used.
  File: ui/FilterTabs.jsx

P3 — VIBES + DISTRICTS TABS ALWAYS EMPTY (stubs)
  case 'posts'/'groups': return [Promise.resolve([])]
  EmptyState shown to user on every search in these tabs. No UX indication they're unimplemented.
  File: dal/search.dal.js:63-66

P3 — CitizensRow + VportsRow contain only mock/static data
  CitizensRow renders Citizen_1..5 static placeholders.
  VportsRow renders 3 hardcoded fake VPORTs.
  These are wired to ExploreFeed which is off — not a live bug, but blocks enablement.
  Files: ui/CitizensRow.jsx, ui/VportsRow.jsx
