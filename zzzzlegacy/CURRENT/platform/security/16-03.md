### RLS Policy — Manual SQL Required

The user must run this SQL in Supabase SQL Editor to fix the `vport.profiles` UPDATE RLS policy:

```sql
-- 1. Review existing policies (run first, verify before step 2)
SELECT
  pol.polname,
  CASE pol.polcmd
    WHEN 'r' THEN 'SELECT'
    WHEN 'a' THEN 'INSERT'
    WHEN 'w' THEN 'UPDATE'
    WHEN 'd' THEN 'DELETE'
    ELSE 'ALL'
  END AS command,
  pg_get_expr(pol.polqual, pol.polrelid)       AS using_expr,
  pg_get_expr(pol.polwithcheck, pol.polrelid)  AS with_check_expr
FROM pg_policy pol
JOIN pg_class cls ON cls.oid = pol.polrelid
JOIN pg_namespace nsp ON nsp.oid = cls.relnamespace
WHERE nsp.nspname = 'vport'
  AND cls.relname = 'profiles';

-- 2. Add UPDATE policy using vc.actor_owners ownership model
--    Drop the old UPDATE policy first if one exists, then create:
CREATE POLICY "profiles_update_owner"
ON vport.profiles
FOR UPDATE
TO authenticated
USING (
  EXISTS (
    SELECT 1
    FROM vc.actor_owners ao
    WHERE ao.actor_id = vport.profiles.actor_id
      AND ao.user_id = auth.uid()
      AND coalesce(ao.is_void, false) = false
  )
)
WITH CHECK (
  EXISTS (
    SELECT 1
    FROM vc.actor_owners ao
    WHERE ao.actor_id = vport.profiles.actor_id
      AND ao.user_id = auth.uid()
      AND coalesce(ao.is_void, false) = false
  )
);
```

Note: If a conflicting UPDATE policy already exists, drop it first:
`DROP POLICY "old_policy_name" ON vport.profiles;`
