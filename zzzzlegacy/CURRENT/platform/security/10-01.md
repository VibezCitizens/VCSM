# Planning — May / 10 / Sequence 01

## Task: Fix HIGH security blocker — unauthenticated post visibility bypass

### Root Cause (verified by BUGSBUNNY/VENOM/THOR session)
`getPostById.controller.js` line 31: `if (viewerActorId)` guard skips the entire
app-layer visibility gate when the viewer is null (no session / identity not loaded).
Combined with `checkPostVisibilityDAL` failing open for null viewerActorId
(`return { canView: true, reason: "no_context" }`), a null viewer receives
the full post payload for private-actor posts without any app-layer check.

RLS provides a fallback for truly unauthenticated (anon role) requests, but a
logged-in user with an unresolved identity (actorId = null) bypasses the gate.

### Scope: VCSM

### Files Modified

- `apps/VCSM/src/features/post/postcard/dal/postVisibility.dal.js`
  — Restructured to handle null viewerActorId: skip block check, check privacy,
    return `{ canView: false, reason: "private_no_viewer" }` for private actors
- `apps/VCSM/src/features/post/postcard/controller/getPostById.controller.js`
  — Removed `if (viewerActorId)` guard; visibility gate now always runs

### No debug probes built — root cause was proven by static analysis.
