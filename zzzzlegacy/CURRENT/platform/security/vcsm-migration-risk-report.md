## CARNAGE MIGRATION PLAN / FINDING #2 — MISSING RLS POLICY

```
CARNAGE TARGET
Object: vport.profile_categories — Public SELECT policy
Application Scope: VCSM
Type of risk: Missing RLS Policy
```

**Current behavior:** `profile_categories` is queried from the browser in identity, vportPublicDetails, and readVportType DALs. If no public SELECT policy exists, authenticated reads from non-owner actors will silently return null for vport_type.

**Risk:** Public vport profile pages cannot determine vport_type → wrong type shown or features hidden.

**Example SQL proposal (text only — do not run):**
```sql
-- Option A: Allow any authenticated actor to read primary categories
CREATE POLICY "profile_categories_authenticated_read"
  ON vport.profile_categories
  FOR SELECT
  TO authenticated
  USING (true);

-- Option B: Allow public read of primary categories only
CREATE POLICY "profile_categories_public_read"
  ON vport.profile_categories
  FOR SELECT
  TO public
  USING (is_primary = true);
```

**Rollback plan:** RLS policies are additive. DROP POLICY to roll back. No data loss risk.
