## Bug Summary

**Symptom:** `permission denied for table profile_public_details` — settings save and flyer save are blocked despite the table reference being correct (schema: vport, table: profile_public_details, conflict: profile_id).

**Root cause:** RLS is enabled on `vport.profile_public_details` but no write policies exist for the `authenticated` role, and the role may lack INSERT/UPDATE grants.

---

## Execution Plan

### Proposed SQL — vport.profile_public_details RLS

**Step 1:** Enable RLS (idempotent)
**Step 2:** Grant SELECT to anon, SELECT/INSERT/UPDATE to authenticated
**Step 3:** SELECT policy — public read for active non-deleted vports (anon + authenticated)
**Step 4:** INSERT policy — owner only (owner_user_id = auth.uid())
**Step 5:** UPDATE policy — owner only (USING + WITH CHECK)

All policies use inline auth.uid() — no SECURITY DEFINER helper functions.
