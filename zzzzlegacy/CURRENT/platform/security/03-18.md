## Task

Fix VPORT Business Card "Profile" button — TRAZE 404 for VPORTs not in TRAZE dataset

## Root Cause

File: `apps/VCSM/src/features/public/vportBusinessCard/view/VportBusinessCardPublic.view.jsx:66-70`

```js
const profileHref = useMemo(() => {
  const key = card?.slug;
  if (!key) return "";
  return `https://traze.vibezcitizens.com/pro/${key}`;
}, [card?.slug]);
```

Profile button always links to TRAZE using the VCSM slug. TRAZE is a statically generated
Next.js app — only providers explicitly listed in its dataset have pages. VPORTs not in the
TRAZE dataset (e.g., `bar-ber-34b1`) produce a Next.js 404.

Intended behavior: Profile → TRAZE (user confirmed). But 404 must not occur.

Fix strategy: Gate the TRAZE link on `business_card_settings.traze_provider_slug` (a key in the
existing JSONB column — no schema migration). If not set, fall back to the VCSM business card
URL which is always a valid public route.
