# Planning — April / 19 / Sequence 11

**Task:** Fix VPORT menu add-item — new items must appear immediately without page refresh
**Scope:** VCSM
**Task Class:** MIXED (DEBUGGING + IMPLEMENTATION)

## Execution Plan

Root cause: `getVportActorMenuController` has a 60s TTL cache keyed on `actorId` when `includeInactive=false`. The manage panel defaults `includeInactive=false`, so `refresh()` after add/edit/delete returns the stale cached result — overwriting the optimistic insert with old server data.

Fix: import `invalidateMenuCache` from the controller and call it in `itemsMut.onSuccess` before `refresh()` so the post-mutation fetch always hits the DB.

Files Expected To Change:
- `apps/VCSM/src/features/profiles/kinds/vport/screens/menu/components/VportActorMenuManagePanel.jsx`

Execution Type: Direct

## Execution Summary

### What was completed

1. **Cache invalidation fix** — `VportActorMenuManagePanel.jsx`
   - Added import: `invalidateMenuCache` from `getVportActorMenu.controller`
   - Modified `itemsMut.onSuccess` to call `invalidateMenuCache(actorId)` before `refresh()`
   - This ensures add-item, edit-item, and delete-item all trigger a fresh DB fetch on the next `refresh()` call
   - Also resolves the edit-item stale display issue (same code path)

2. **Reviews page profile fallback** — `readVportPublicDetails.rpc.dal.js`

   - `mapVportPublicDetailsRpcResult` already handles these field names — no model changes needed
   - This fix applies to both reviews page and any other public page using `useVportPublicDetails` for actors without menu items
