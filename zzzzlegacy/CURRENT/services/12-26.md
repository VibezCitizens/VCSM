# 12-26

> Feed N+1 Query Fix — batch comment counts and reaction data in the feed pipeline.

## Audit

The feed pipeline (`fetchFeedPage.pipeline.js`) fetches posts in pages of 10 and hydrates actors, media, mentions, blocks, follows, and hidden posts in a single batched `Promise.all`. However, once PostCard components render, **each card fires 3 per-post DB queries** via hooks:

| Hook | File | Table(s) | Queries/card |
|------|------|----------|-------------|
| `usePostCommentCount` | `postcard/ui/PostCard.view.jsx` | `vc.post_comments` | 1 |
| `usePostReactions` | `postcard/components/ReactionBar.jsx` -> `getPostReactionsController` | `vc.post_reactions` (viewer reaction) + RPC `post_reactors_summary_one` (counts) | 2 |

**Total N+1 per page: 30 queries (3 x 10 posts)**. Over multiple pages this compounds to 700+.

## Fix Applied

### New batched DALs (feed layer)
- `feed.read.commentCounts.dal.js` — single query to `vc.post_comments` for all postIds, counted in JS
- `feed.read.viewerReactions.dal.js` — single query to `vc.post_reactions` filtered by viewer actorId
- `feed.read.reactionCounts.dal.js` — two parallel queries (`vc.post_reactions` + `vc.post_rose_gifts`), aggregated in JS

### Pipeline integration
- Added 3 new batch DALs to the existing `Promise.all` in `fetchFeedPage.pipeline.js` (9 parallel DALs, up from 6)
- Results passed through `normalizeFeedRows` and attached to each post row as `commentCount`, `viewerReaction`, `reactionCounts`

### PostCard chain update
- `PostCard.view.jsx` — detects pre-loaded data via `HAS_PRELOADED()`, passes `null` postId to `usePostCommentCount` to skip its fetch
- `ReactionBar.jsx` — accepts `preloadedReaction` and `preloadedCounts` props, passes to `usePostReactions`
- `usePostReactions` — skips initial fetch `useEffect` when `preloadedCounts` is provided; still works normally for non-feed contexts (PostDetail screen)

### Query reduction
- Before: 30 per-post queries per 10-post page
- After: 3 batched queries (run in parallel with existing 6) per 10-post page
- Net reduction: **~90% fewer DB round-trips for reactions and comments**
