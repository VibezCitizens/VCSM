## Architecture Notes

- Supabase client: `supabase.schema("reviews").from(...)` — same pattern as vport schema DALs
- No RPCs involved — all view reads
- Auth check for write CTA: `supabase.auth.getSession()` in a lightweight effect inside the panel
- No new routes needed — reviews live as a tab inside `/profile/:slug/menu`
- Write review CTA: routes to `/login` if unauthenticated. If authenticated, shows CTA but marks it as future (no submission UI in this task)
- TTL cache on summary + dimensions (60s); no cache on paginated list
