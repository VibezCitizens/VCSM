## Execution Summary

### What was completed

1. **vport.bookings INSERT RLS gap fixed** — Created migration `20260426020000_vport_bookings_insert_rls.sql` adding `bookings_insert_public_pending` (citizens, status=pending, source=public) and `bookings_insert_owner` (profile managers, management sources) policies.

2. **vport.resources.profile_id added to DAL/model** — `vportResource.read.dal.js` select and `VportResource.model.js` mapper updated to include `profile_id`. Required for `vport.bookings` INSERT (RLS checks `profile_id`).

3. **vport booking DAL created** — `vportBooking.read.dal.js` and `vportBooking.write.dal.js` added. Cover `dalGetVportBookingById`, `dalListVportBookingsInRange`, `dalListVportBookingsByResource`, `dalInsertVportBooking`, `dalUpdateVportBookingStatus`.

4. **vport availability DAL created** — `vportAvailability.read.dal.js` and `vportAvailability.write.dal.js` added. Cover `dalListVportAvailabilityRulesByResourceId`, `dalListVportAvailabilityExceptionsInRange`, `dalUpsertVportAvailabilityRule`, `dalUpsertVportAvailabilityException`.

5. **createBooking.controller dual-routed** — Tries `dalGetVportResourceById` first. If vport resource: validates `profile_id`, permission-checks via `assertActorCanManageResource`, inserts into `vport.bookings`. If vc resource: existing flow via `assertActorOwnsVportActor`, inserts into `vc.bookings`.

6. **getResourceAvailability.controller dual-routed** — Same pattern: vport resource → reads `vport.availability_rules`, `vport.availability_exceptions`, `vport.bookings`. vc resource → existing vc tables + service profiles.

7. **setAvailabilityRule.controller dual-routed** — vport resource → `assertActorCanManageResource` + `dalUpsertVportAvailabilityRule`. vc resource → `assertActorOwnsVportActor` + `dalUpsertAvailabilityRule`.

8. **setAvailabilityException.controller dual-routed** — Same pattern as setAvailabilityRule.

9. **BookingAvailability.model extended** — `mapResourceAvailabilityModel` accepts `isVportResource` flag; if true, maps resource via `mapVportResourceRow`.

10. **QR is_active fix** — `qrLink.read.dal.js` adds `is_active` to `QR_SELECT` + `.eq('is_active', true)` in slug lookup. `QrLink.model.js` maps `isActive` field.

11. **createQrLink permission guard** — Full 4-level fallback chain: org → location → resource → profile. `requestActorId` now required.

12. **mountedRef guards** — Added to `useOrganizationWorkspace`, `useOrganizationLocations`, `useLocationResources`, `useResourceServiceOverrides`.

13. **Logan documentation sync** — See Phase H below.
