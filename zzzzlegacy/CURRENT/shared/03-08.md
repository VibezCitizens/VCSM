### Slice 4 — Root Cause + Fix Plan
ROOT CAUSE CONFIRMED: Temporal ordering — noti:refresh invalidates badge at T0 (before inbox has marked items seen). Badge refetches with pre-mark-seen state and caches stale N count. Inbox fetch completes at T2 marking all items seen but has no path to re-invalidate badge RQ cache. Badge shows old count until next poll (60s) or next navigation (noti:refresh).

Second navigation works because items are already seen in DB when badge refetches.

Fix 1 (CRITICAL): Add useEffect in useNotificationInbox.js watching query.dataUpdatedAt → invalidate notificationUnread after each successful inbox fetch. This fires post-mark-seen, returning correct 0 count.

Fix 2 (MEDIUM): Remove internal noti:refresh dispatch from markAllNotificationsSeen — onSuccess handles invalidation.

Fix 3 (LOW): Add purgeNotificationCache() to bootstrap.invalidate.js, call on actor switch alongside purgeChatMessageCache().
