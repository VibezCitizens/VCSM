## Execution Summary

### Root causes found and fixed

**Bug 1 — Overall score shows "-" with 0 ratings (CONFIRMED, FIXED)**
`VportReviewsView.jsx` read `r.stats?.officialOverallAvg` but the hook returns `r.officialStats` (not `r.stats`). Fixed to use `r.overallAverage` and `r.totalReviews`.

**Bug 2 — Author "Anonymous" (CONFIRMED, FIXED)**
Root cause: RLS on `vc.actors` blocks private user actors from non-owners/non-followers. When `dalListActorCardsByActorIds` queries `vc.actors`, the author's row is silently filtered out by `can_view_actor()` because the author is private and the viewer doesn't follow them. The enrichment returns empty → falls back to "Anonymous".

Fix: Added fallback to `identity.actor_directory` table. When `vc.actors` returns empty for some author IDs (RLS-blocked), the DAL now queries `identity.actor_directory` for those missing IDs to get display_name, username, avatar_url. This table is designed for cross-context lookups and has more permissive access.

**Bug 3 — Avatar empty string (CONFIRMED, FIXED)**
`dalListActorCardsByActorIds` returned `avatarUrl: ""` instead of null for missing photos. Empty string is truthy so the UI's `|| "/avatar.jpg"` fallback never triggered. Fixed to return null.

### Files changed
- `apps/VCSM/src/features/profiles/kinds/vport/screens/review/VportReviewsView.jsx` — fixed overall score property access
- `apps/VCSM/src/features/profiles/kinds/vport/dal/review/vportReviewAuthors.read.dal.js` — added identity.actor_directory fallback for RLS-blocked actors, fixed avatar empty string, improved displayName fallback

## Execution Summary

### Root causes found and fixed

**Bug 1 — Missing overall score (CONFIRMED, FIXED)**
`VportReviewsView.jsx` line 82-83 read `r.stats?.officialOverallAvg` but the hook returns `r.officialStats` (not `r.stats`). `r.stats` was always `undefined`, so the score always showed `"-"`. Fixed to use `r.overallAverage` and `r.totalReviews` (the hook's computed values), with `r.officialStats` as fallback.

**Bug 2 — Author avatar empty string (CONFIRMED, FIXED)**
`dalListActorCardsByActorIds()` returned `avatarUrl: p.photo_url ?? ""` — empty string is truthy, so the UI's `|| "/avatar.jpg"` fallback never triggered. Fixed to return `avatarUrl: p.photo_url || null` so null/empty both fall through to the avatar fallback. Also improved `displayName` to try `username` before falling back to "Anonymous".

**Bug 3 — Author "Anonymous" (LIKELY — data issue)**
If the author actor's `profile_id` is null in `vc.actors`, or the corresponding `public.profiles` row is missing/has null display_name, the DAL returns "Anonymous". This is a data-level issue that requires DB verification. The code fallback path is now improved (tries username before "Anonymous"), but the underlying data issue may need a profile_id fix on the actor row.

### Files changed
- `apps/VCSM/src/features/profiles/kinds/vport/screens/review/VportReviewsView.jsx` — fixed overall score property access
- `apps/VCSM/src/features/profiles/kinds/vport/dal/review/vportReviewAuthors.read.dal.js` — fixed avatar empty string fallback, improved displayName fallback chain
