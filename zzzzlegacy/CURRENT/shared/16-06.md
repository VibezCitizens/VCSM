## Execution Summary

**Approved:** 2026-04-16 — user provided full spec equivalent to EXECUTE

**Root Cause:** Three SECURITY DEFINER functions in the `reviews` schema (`validate_target_actor_is_active_vport`, `resolve_target_subtype`, `hydrate_review_snapshots`) still referenced the dropped `vc.vports` relation. `upsert_neutral_review` calls `validate_target_actor_is_active_vport` at entry point, causing immediate failure.

**Fix:** Three `CREATE OR REPLACE FUNCTION` statements. No signature changes. No trigger renames. No app code changes.

**Key mapping applied:**
- `JOIN vc.vports WHERE actor_id = p_actor_id` → `JOIN vport.profiles vp WHERE vp.actor_id = p_actor_id`
- `v.is_active = true` → `vp.is_active = true AND vp.is_deleted = false`
- `v.vport_type` → `vport.profile_categories.category_key WHERE is_primary = true`
- Snapshot columns (`name`, `slug`, `avatar_url`) → read from `vport.profiles`
- Author snapshot path: only overwritten when author has a `vport.profiles` row (user-author path preserved)
- Snapshot target columns resolved from `reviews.write.dal.js` REVIEW_RETURN_COLUMNS: `target_display_name_snapshot`, `target_username_snapshot`, `target_avatar_url_snapshot`, `author_display_name_snapshot`, `author_username_snapshot`, `author_avatar_url_snapshot`

**Data fix note:** Legacy `vc.vports.vport_type` stored `'gas station'` (with space). `vport.profile_categories.category_key` uses `[a-z0-9_]` constraint — must be `'gas_station'`. Separate data-fix migration provided:
```sql
UPDATE reviews.review_dimensions
SET target_subtype = replace(target_subtype, ' ', '_')
WHERE target_subtype LIKE '% %';
```
Must be run only if the audit query confirms spaced values exist.

**Files Changed:** DB only — no JS files modified.

**Residual audit:** Run `SELECT routine_name FROM information_schema.routines WHERE routine_schema = 'reviews' AND routine_definition ILIKE '%vc.vports%'` — expected 0 rows post-migration.
