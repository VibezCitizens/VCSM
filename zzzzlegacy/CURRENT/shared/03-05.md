## Execution Plan

Task: Fix slow chat open transition — Inbox → ConversationView optimistic shell
Application Scope: VCSM

Files Expected To Change:

Specialist Commands Required: None
SENTRY Review: Optional / Post-Execution
SENTRY Timing: Post-Execution

Root Cause:
ConversationView.jsx had a blocking gate — `if (loading || !conversation || !members?.length)`
that rendered a full-page spinner until `useConversation` + `useConversationMembers` both
resolved. `openConversation.rpc.js` performs 3–5 serial DB round trips (membership check,
optional reactivate, inbox_entries check, optional insert, SELECT conversations), causing a
visible ~4 second "Loading conversation…" block on every chat open.

Fix:
Two-part fix — VCSM only, no engine changes:
1. InboxScreen passes the matching inbox preview object via React Router navigation state
2. ConversationView reads `location.state?.inboxPreview`, builds `seedConversation` +
   `seedPartnerActor` memos from it, uses `effectiveConversation = conversation ?? seedConversation`
   and `effectivePartnerActor = resolvedPartnerActor ?? seedPartnerActor`. Full-page spinner
   only fires when `!effectiveConversation` (truly no data). Message list shows animated skeleton
   while `messagesLoading && messages.length === 0`. DEV timing marks added via `chatNavDbg`.
