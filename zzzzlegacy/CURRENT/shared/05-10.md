### Proven Root Cause

The Invalid Hook Call is most likely caused by a **stale Vite dev server cache** combined with the engine alias boundary. When engine hooks import React, Vite must resolve them through the same pre-bundled React module. The `dedupe` directive (added in 05-08) was the primary fix. This pass adds two hardening measures:

1. `framer-motion` added to dedupe list (prevents any edge case where AnimatePresence gets a separate framer-motion instance)
2. `optimizeDeps.include: ['react', 'react-dom', 'framer-motion']` — explicitly forces pre-bundling even with `entries: []`, ensuring these critical packages are always in the pre-bundle graph
