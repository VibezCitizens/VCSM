### Slice 1 — useChatMessagePrefetch + InboxScreen wiring

**Created:** `apps/VCSM/src/features/chat/inbox/hooks/useChatMessagePrefetch.js`

- `useQueryClient()` — shares the app's existing React Query client
- `idsKey` = top-10 conversation IDs joined as a string — effect dependency is stable even though `conversationIds` array reference changes each render
- For each ID: if `queryClient.getQueryData(queryKeys.chatMessages(cid))` exists → log prefetch:hit, skip
- Otherwise: call `queryClient.prefetchQuery({ queryFn: getConversationMessages(...), staleTime: 90_000 })`
- Concurrency pool: starts 3 fetches, each `.then()` fires `runNext()` to maintain waterfall at depth 3
- `pendingRef` tracks in-flight IDs across renders to prevent duplicate concurrent fetches
- DEV marks: `prefetch:hit`, `prefetch:start`, `prefetch:miss`

**Modified:** `apps/VCSM/src/features/chat/inbox/screens/InboxScreen.jsx`

### Slice 2 — VCSM useConversationMessages cache wrapper

**Modified:** `apps/VCSM/src/features/chat/conversation/hooks/conversation/useConversationMessages.js`

- Added `useQueryClient` import
- `cached = queryClient.getQueryData(queryKeys.chatMessages(conversationId)) ?? []` — synchronous read at render time
- `servedFromCacheRef = useRef(cached.length > 0)` — captured at mount
- Engine hook still called with same params — unchanged
- Write-back effect: when `!engine.loading && engine.messages.length > 0` → `queryClient.setQueryData(...)` keeps cache fresh after every load and realtime update
- `messages` = `(engine.loading && engine.messages.length === 0) ? cached : engine.messages`
- `loading` = `engine.loading && cached.length === 0 && engine.messages.length === 0`
- Returns `{ ...engine, messages, loading }` — all write callbacks unchanged

### Cache Lifecycle
- **staleTime**: 90s — `prefetchQuery` won't refetch data that was fetched within the last 90s
- **gcTime**: React Query default (5 min) — cache entries evicted 5 min after last subscriber unmounts
- **Write-back**: engine resolves → `setQueryData` → next open of same conversation is warm
- **Actor switch / logout**: caller must `queryClient.invalidateQueries({ queryKey: ['chat', 'messages'] })` or `queryClient.clear()` — noted in file comment
