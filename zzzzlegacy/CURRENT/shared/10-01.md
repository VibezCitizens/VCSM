## Execution Summary

### Root Cause

The `<h1>` tags inside `TermsOfServiceContent.jsx` and `PrivacyPolicyContent.jsx` were hardcoded with `v1.0`. The `LegalDocumentScreen` rendered these components directly without overriding the title — so regardless of what version the DB had or what `?v=` param was in the URL, the page always displayed `v1.0`.

### Fix Applied (earlier in this session)

**1. Removed hardcoded titles from content files:**
- `features/legal/docs/TermsOfServiceContent.jsx` — removed `<h1>Vibez Citizens Terms of Service v1.0</h1>` and `Effective Date` line
- `features/legal/docs/PrivacyPolicyContent.jsx` — removed `<h1>Vibez Citizens Privacy Policy v1.0</h1>` and `Effective Date` line

**2. Added DAL function to resolve specific document:**
- `features/legal/dal/legalDocuments.read.dal.js` — added `dalGetLegalDocument({ appKey, documentType, version })` which fetches a specific version if `?v=` is provided, or falls back to the active version

**3. Updated screen to render dynamically from DB:**
- `features/legal/screens/LegalDocumentScreen.jsx` — reads `?v=` from `useSearchParams()`, calls `dalGetLegalDocument()` on mount, renders `<h1>{title} v{version}</h1>` + effective date from `published_at`
