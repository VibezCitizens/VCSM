**Root cause (BUGSBUNNY):**
1. `inbox.read.dal.js` doesn't select `partner_display_name` from `inbox_entries` — the schema has this column but the query skips it
2. `inbox.read.dal.js` doesn't select `title` from the `conversations` embed
3. When actor summary hydration returns no results (missing profile), there's zero fallback source — model returns null → UI shows "Unknown"

## Execution Summary

**What was completed:**
- Added `partner_display_name`, `partner_username`, `partner_photo_url` to the inbox_entries SELECT in `inbox.read.dal.js`
- Added `title` to the conversations embed SELECT in the same query
- Updated `InboxEntryModel` to fall back to `raw.partner_display_name`, `raw.partner_username`, `raw.partner_photo_url` when member hydration yields no display name

**Why:** The inbox query was skipping 3 partner-name columns that exist in the schema and 1 conversation title column. When actor summary hydration returned no match (missing profile), the model had zero fallback — always null → UI rendered "Unknown". Now the model reads the partner name directly from `inbox_entries` as a fallback.
