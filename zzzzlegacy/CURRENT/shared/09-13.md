### What was done
Added `_switchVersion` guard to `switchActor()` in `identityContext.jsx`. This was the last remaining violation of the Single-Source Actor Architecture contract (Rule 5).

### How it works
1. `let _switchVersion = 0` declared at module level (line 23) alongside existing `_resolveVersion`
2. `const mySwitchVersion = ++_switchVersion` captured at start of every `switchActor()` call (line 31)
3. After async hydration completes, check `mySwitchVersion !== _switchVersion` — if a newer switch started while we were resolving, abort with `SWITCH_ABORT_STALE` debug event
4. Only the most recent switch commits via `setIdentity()`

### Race condition now prevented
```
switchActor(A)  v1 starts
switchActor(B)  v2 starts
switchActor(C)  v3 starts

A resolves → v1 !== v3 → ABORT (stale)
B resolves → v2 !== v3 → ABORT (stale)
C resolves → v3 === v3 → COMMIT ← correct
```
