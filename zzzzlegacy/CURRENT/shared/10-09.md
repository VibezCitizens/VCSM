### Existing System-Post Pattern (Gas/Exchange/Menu)
- `resolvePublicRealmIdDAL()` — synchronous, returns `PUBLIC_REALM_ID = "2d6c267f-9c43-48e4-aa5e-e0a0274e9bc2"` from constant
- Dedup DAL: queries `vc.posts` by `actor_id + post_type + created_at >= since`
- Name resolver DAL: queries `vport.profiles.name` by `actor_id`
- `createSystemPost` from `features/upload/adapters/posts.adapter.js`

### Save Flow Map
| Feature | Save Entry Point | Component | Hook | Controller |
|---|---|---|---|---|
| Service Area (create) | `VportDashboardLocksmithScreen.jsx` | `AreaForm` | `useLocksmithOwner.addArea` | `ctrlAddServiceArea` |
| Service Area (update) | `VportDashboardLocksmithScreen.jsx` | `AreaForm` | `useLocksmithOwner.updateArea` | `ctrlUpdateServiceArea` |
| Service Details | ❌ No UI save point exists | — | `useLocksmithOwner.saveServiceDetail` (defined, never called) | `ctrlSaveServiceDetail` |
| Availability Hours | `WeeklyAvailabilityGrid` → "Save Working Hours" button | `WeeklyAvailabilityGrid.save()` | `useVportManageAvailability` | `manageVportAvailabilityRuleController` |
| Portfolio (create) | `PortfolioItemForm.jsx` → `handleSubmit` | `PortfolioItemForm` | `usePortfolioItemSubmit` | `ctrlSavePortfolioDetail` + engine |

### Ownership Gate
- All three screens (`VportDashboardLocksmithScreen`, `VportDashboardCalendarScreen`, `VportDashboardPortfolioScreen`) already gate to `isOwner` before rendering content
- No new ownership check needed — checkboxes are only visible inside owner-gated views

### Feed Renderer Audit
- `PostCard.view.jsx` has special renderers for `fuel_price_update` (FuelPriceBoard) and `menu_update` (view menu link)
- All other post_types fall into the `else` block → renders `text` + optional `media` → **safe for all 4 locksmith post_types**

### No RLS Changes Required
- `createSystemPost` uses `supabase.auth.getUser()` for auth; locksmith owners are authenticated actors
- Locksmith owners already have `INSERT` on `vc.posts` per existing RLS (same as exchange/gas/menu owners)

### Service Details — Deferred
- `ctrlSaveServiceDetail` is defined in `locksmithOwner.controller.js` and exposed in `useLocksmithOwner` but has **no UI save point**
- `ServiceDetailRow` and `GapServiceRow` in `VportDashboardLocksmithScreen` are read-only
- `locksmith_service_update` post_type is deferred until a service detail editor is built

### Hours — Approach Note
- `WeeklyAvailabilityGrid` is shared across all VPORT types
- It has a single batch "Save Working Hours" button with a `save()` function
- Adding an `onSaved` callback prop (optional, no default) to `WeeklyAvailabilityGrid` is minimal and non-breaking
- Locksmith hours feed share is activated by:
  - `VportDashboardCalendarScreen.jsx` reading `identity?.vportType === 'locksmith'`
  - Showing a checkbox above the grid: "Share your updated hours to my feed"
  - Passing `onSaved` callback that conditionally calls publish controller
