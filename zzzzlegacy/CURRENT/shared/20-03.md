## Execution Summary

### Root Cause

`apps/VCSM/src/features/chat/conversation/components/MessageBubble.jsx` lines 108–121 (pre-fix).

The avatar slot had two branches:
- `!isMine && showAvatar` → rendered `<div className="shrink-0 w-7">` (avatar slot for last message in group) — when `actor` is null, this was an empty 28px div.
- else `!isMine` → always rendered `<div className="w-7" />` (spacer for non-last messages) — unconditional, no guard on actor presence.

When actor hydration returns null (no sender actor data for incoming messages), BOTH branches produced a 28px empty div before every incoming bubble, creating the phantom left gutter.

### Fix Applied

Gated the non-last-message spacer on `actor` being non-null. When actor is null, `null` is rendered instead of the spacer div — the bubble collapses to the left edge.

Three-way logic:
- `showAvatar = true` → always render avatar slot (avatar shows if actor exists; slot reserved even if actor null for future when actor loads)
- `showAvatar = false, actor present` → render `w-7` spacer (alignment within group — avatar will appear on last message)
- `showAvatar = false, actor null` → render `null` (no spacer — collapse to left edge)

This is future-safe: when actor hydration is wired, the spacer correctly reappears for non-last messages and the avatar renders on the last.
