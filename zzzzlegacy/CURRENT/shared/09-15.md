### Controller Migration (1 file — highest risk)
- `apps/VCSM/src/features/profiles/kinds/vport/controller/review/VportReviews.controller.js` — Full rewrite

Controller mapping decisions:
- Engine `key` → app `dimensionKey` (dimensions)
- Engine `reviewCount` → app `totalReviews` (stats)
- Engine `overallAvg` → app `overallAverage` (stats)
- Engine `authorCard` object → app flat `authorDisplayName/authorUsername/authorAvatarUrl` (reviews)
- Engine `authorDisplayNameSnapshot` used as fallback when authorCard is null
- Submit flow: loads engine dimensions to build `dimensionKey → dimensionId` map, then maps incoming ratings
- `ctrlAssertReviewTargetActor` stays app-local (queries vc.actors, outside engine scope)
- `ctrlDeleteMyReview` supports both positional and object call styles (hook calls with positional)
- `listReviews` derives `hasMore` from `reviews.length >= limit` since engine returns `nextCursor` only
