### Root Cause

Missing `resolve.dedupe` in vite.config.js. When Vite pre-bundles or optimizes dependencies, engine files outside the app root directory can get a **separate module instance** of React. This causes `useContext` to find a null dispatcher because the engine hooks use a different React than the app components.

### Fix Applied

Added `dedupe: ['react', 'react-dom']` to `resolve` block in vite.config.js. This forces Vite to always resolve React to the single instance in the app's node_modules.

### Files Modified
- apps/VCSM/vite.config.js (added `dedupe: ['react', 'react-dom']`)
