# 12-23 — Notification Legacy Cleanup

**Application Scope:** VCSM
**Task Class:** IMPLEMENTATION
**Command:** Wolverine
**Status:** COMPLETE

## Summary

Audited and removed all legacy `vc.notifications` code paths from the VCSM app notification system. The notification engine migration is now fully reflected in app code.

## What Was Done

### Phase 1 — Audit

Searched `apps/VCSM/src/` for all references to:
- `vc.notifications` / `from('notifications')` in vc schema
- `dalInsertNotification`, `markNotificationRead`, `markAllNotificationsSeenDAL`
- Legacy DAL file imports

Found legacy references in:
- `notifications.create.dal.js` — legacy insert via RPC/direct insert
- `notifications.write.dal.js` — legacy mark-read/mark-seen
- `inboxUnreadCount.dal.js` — orphaned (queries `chat.inbox_entries`, zero importers)
- `notificationsFeature.group.js` — dev diagnostics importing all three legacy DALs + inline `vc.notifications` queries
- `professionalBriefings.read.dal.js` — still reads `vc.notifications` (separate feature, out of scope)

### Phase 2 — Removal

**Deleted files:**
- `apps/VCSM/src/features/notifications/inbox/dal/notifications.create.dal.js`
- `apps/VCSM/src/features/notifications/inbox/dal/notifications.write.dal.js`
- `apps/VCSM/src/features/notifications/inbox/dal/inboxUnreadCount.dal.js`

**Updated file:**
- `apps/VCSM/src/dev/diagnostics/groups/notificationsFeature.group.js`
  - Replaced `dalInsertNotification` with `publishVcsmNotification` from `@/features/notifications/publish`
  - Replaced `markNotificationRead` with `markRead` from `@notifications` engine
  - Replaced inline `vc.notifications` query in `resolve_senders_mapper` test with `getNotifications` controller
  - Removed `latestDiagnosticsNotification` helper (direct `vc.notifications` query)
  - Removed unused `supabase` import

**Kept files (still needed):**
- `senders.read.dal.js` — used by `resolveSenders.js` (active in controllers + diagnostics)
- `blocks.read.dal.js` — used by `blockFilter.js` (active in controllers)

### Phase 3 — Verification

Post-removal grep confirms:
- Zero active `vc.notifications` code references in notification feature code
- Zero imports of deleted DAL files
- Only remaining `vc.notifications` references are migration comments in controllers (no code) and `professionalBriefings.read.dal.js` (separate feature, separate migration)

## Out of Scope (Noted for Future)

- `professionalBriefings.read.dal.js` still reads `vc.notifications` — needs its own migration task
- `vc.notifications` database table removal — separate DB migration (Carnage task)
