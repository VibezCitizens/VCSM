## Execution Summary

### What was completed

All 7 items executed across ENGINE + VCSM app:

**ENGINE — `engines/chat/src/hooks/useConversationMessages.js`**
- Extended `addOptimistic` to accept `__uploading` flag and optional external `clientId` (for media upload placeholder flow)
- Added `markFailed(clientId)` — keeps message visible with `__failed: true, __optimistic: false` instead of removing it
- Added `retryMessage(clientId)` — resets to pending, re-runs `sendMessageController`, falls back to `markFailed` on second failure
- Changed `onSendMessage` error path: calls `markFailed` instead of `removeOptimistic`
- Added `prebuiltClientId` param to `onSendMessage` — skips `addOptimistic` when a media placeholder was pre-inserted
- Exported `addOptimistic`, `markFailed`, `retryMessage` from hook return value

**VCSM — `useSendMessageActions.js`**
- Accepts `addOptimistic` and `markFailed` params
- `handleAttach` now pre-inserts an `__uploading: true` optimistic placeholder before upload begins
- On upload fail: calls `markFailed(placeholderClientId)` — placeholder stays visible as failed
- On send fail: calls `markFailed(placeholderClientId)` — same
- Passes `prebuiltClientId` to `onSendMessage` to reuse the placeholder

**VCSM — `MessageBubble.jsx`**
- Added `onRetry` prop
- `__uploading` → renders 144×144 placeholder square + "Uploading…" label when no mediaUrl yet
- `__optimistic && !__failed` → row opacity 0.65 + "Sending…" label below bubble
- `__failed` → row opacity 0.80 + "Failed · Retry" tap target calling `onRetry`

**VCSM — `MessageGroup.jsx`**
- Wrapped with `React.memo` — prevents re-render of groups that haven't changed
- Accepts and passes `onRetry` to each `MessageBubble`

**VCSM — `MessageList.jsx`**
- Fixed group key: `key={`group-${group.messages[0]?.id ?? group.messages[0]?.clientId ?? index}`}` — stable across new message arrivals
- Accepts `retryMessage` prop and passes as `onRetry` to each `MessageGroup`

**VCSM — `ConversationView.jsx`**
- Extracts `addOptimistic`, `markFailed`, `retryMessage` from `useConversationMessages`
- Passes `addOptimistic` and `markFailed` to `useSendMessageActions`
- Passes `retryMessage` to `MessageList`
- Smart scroll: auto-scroll only fires when user is ≤100px from the bottom
- Scroll listener updates `showJumpButton` state when distance from bottom exceeds 120px
- "Jump to latest ↓" button renders as floating pill above ChatInput when user has scrolled up; clicking scrolls smooth and hides the button

**VCSM — `InboxScreen.jsx`**
- Replaced bare error div with centered retry UI using `refresh` from `useInbox`
- "Try again" button calls `refresh()` directly — no reload
