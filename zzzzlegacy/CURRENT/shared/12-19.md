## Execution Summary

### What Was Completed

All 5 slices executed. All 7 remaining post-interaction events now publish through the notification engine.

**Self-notification prevention:** Added `actorId === recipientActorId` skip to `publishVcsmNotification()` adapter — every caller gets it automatically.

**Batch publish:** Added `publishVcsmNotificationBatch()` for mention notifications (multiple recipients from one event).

**Slice 1 — Reactions (like/dislike):**
- `togglePostReaction.controller.js` — tracks `created` boolean
- Publishes `social.post.like` or `social.post.dislike` only on NEW reaction (not toggle-off, not switch)
- Resolves post owner via `fetchPostByIdDAL(postId)` (single-row PK read)
- Self-notification: skipped by adapter

**Slice 2 — Rose:**
- `sendRose.controller.js` — always a creation, always publishes
- Publishes `social.post.rose` with `{ qty }` context
- Resolves post owner via `fetchPostByIdDAL(postId)`

**Slice 3 — Comments:**
- `postComments.controller.js` — `createRootComment`: publishes `social.post.comment` to post owner
- `createReplyComment`: publishes `social.post.comment_reply` to parent comment author
- Parent comment author resolved via inline `post_comments` query by `parentCommentId` (single-row PK)
- Both include truncated `body` in context (first 120 chars)

**Slice 4 — Comment like:**
- `commentReactions.controller.js` — publishes `social.post.comment_like` only on new like (not unlike)
- Resolves comment author + post_id via inline `post_comments` query
- linkPath points to `/post/{post_id}`

**Slice 5 — Mention:**
- `createPostController.js` — publishes `social.post.mention` after mention rows inserted
- Uses `publishVcsmNotificationBatch()` since mentions have multiple recipients
- Recipient IDs already available from mention resolution — zero extra DB reads
- Publish call is in the controller (orchestration layer), NOT the DAL — per user guidance

### Implementation Rules Enforced

| Rule | How Enforced |
|------|-------------|
| Publish only on create | `created` boolean in reaction toggle; only `likeComment` branch in comment like; rose is always creation |

| Minimal recipient lookup | All reads are single-row PK: `fetchPostByIdDAL(postId)`, `post_comments.eq('id', parentCommentId)` |
| Mention publish outside DAL | Publish call in `createPostController.js`, not `insertPostMentions.js` |
