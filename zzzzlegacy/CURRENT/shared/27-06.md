### Production bugs found and fixed

**BUG 1 — cancelBooking: notification linkPath exposes raw actor UUID (CRITICAL)**
- File: `engines/booking/src/controller/cancelBooking.controller.js`
- Was: `linkPath: \`/profile/${resource?.owner_actor_id ?? ''}?tab=book\``
- `owner_actor_id` is a raw UUID → platform invariant violation (no raw UUIDs in public URLs)
- Fix: resolved canonical profile_slug via new DAL function; linkPath is omitted when slug is unavailable
- New code: `linkPath: ownerSlug ? \`/profile/${ownerSlug}?tab=book\` : undefined`

**BUG 2 — createBooking: durationMinutes > 1440 not validated**
- File: `engines/booking/src/controller/createBooking.controller.js`
- Was: only `!durationMinutes` check (falsy guard) — 1441 passed through
- Fix: added `if (durationMinutes > 1440) throw` before any resource lookup

**BUG 3 — createBooking: durationMinutes <= 0 not validated**
- File: `engines/booking/src/controller/createBooking.controller.js`
- Was: `!(-30)` is false → negative values passed through without error
- Fix: added `if (typeof durationMinutes !== 'number' || durationMinutes <= 0) throw`

**BUG 4 — createBooking: unknown booking source not rejected**
- File: `engines/booking/src/controller/createBooking.controller.js`
- Was: unknown source (e.g. 'webhook', 'api') skipped all ownership/citizen checks → booking inserted
- Fix: source allowlist validated immediately after required-field checks
- `const ALL_SOURCES = new Set([...MANAGEMENT_SOURCES, ...CITIZEN_SOURCES])` — throws on any unlisted source

**Supporting DAL addition**
- `engines/booking/src/dal/actor.read.dal.js` — added `dalGetVportProfileSlugByActorId({ actorId })`
  Returns `profile_slug` from `vport.profiles` via vportClient. Never throws — returns null on any error.
  Used exclusively by cancelBooking to safely resolve the linkPath slug.
