## Execution Summary

### Slice 1 — useVportNotificationBadges.js (NEW)
- Created `apps/VCSM/src/features/settings/vports/hooks/useVportNotificationBadges.js`
- Uses `useQueries` to fetch `getUnreadNotificationCount(actorId)` in parallel for all active VPORT actor_ids.
- Query key per VPORT: `queryKeys.notificationUnread(actorId)` = `['notifications', 'unread', actorId]` — same key used by the global badge. Cache is shared; switching into a VPORT after visiting Settings hits warm cache immediately.
- `staleTime: 60_000`, `gcTime: 300_000` — matches existing notification poll cadence.
- Returns a render-time lookup function `(actorId) => number`. No memoization wrapper needed — function is only called during JSX render.
- Purge safety: `purgeNotificationCache()` (bootstrap.invalidate.js) removes `['notifications']` prefix on actor switch and logout, covering all per-VPORT counts. No additional wiring required.

### Slice 2 — VportsTab.view.jsx (MODIFIED)
- Imported `useVportNotificationBadges`.
- Called with `activeVports` after the `activeVports`/`deactivatedVports` split.
- Per active row: extracted `vportActorId = resolveVportActorId(v)` (consistent with `isActive` check), called `getVportCount(vportActorId)`.
- Wrapped button in `<div className="flex shrink-0 items-center gap-2">` to hold badge + button together.
- Badge: `<span>` with `bg-purple-500/90`, renders only when `unreadCount > 0`, caps display at `99+`.

### Category Badge Audit
`countUnread({ recipientActorId })` returns total unread — no category breakdown.
To implement per-category counts (booking / review / social / follow) requires a new runtime function:
  `countUnreadByKind(actorId)` → `{ booking: N, review: N, social: N, follow: N }`
This would query `notification.inbox_items JOIN notification.events` grouping by `event_key` prefix. No DB schema change needed. Capture as a future task.
