### How It Works

1. **Optimistic state inserted** — Before the server call, a temporary review object is built from identity context + form state with `id: "optimistic-" + crypto.randomUUID()` and `isOptimistic: true`. It is prepended to `activeList` via `setActiveList(prev => [optimisticReview, ...prev])` and set as `myReview`.

2. **Reconciliation** — When the server responds successfully, the optimistic entry is replaced in-place: `prev.map(r => r.id === optimisticId ? { ...saved, isOptimistic: false } : r)`. `myReview` is updated to the real server response. Stats reload non-blocking via `loadCore()`.

3. **Failure rollback** — If the request fails, the optimistic entry is removed: `prev.filter(r => r.id !== optimisticId)`. `myReview` reset to null, `myExists` reset to false, error state set.

4. **Sorting preserved** — Optimistic review is prepended (index 0) with `reviewActivityAt: now()`, which is the most recent timestamp, matching the `review_activity_at DESC` sort order.

### What Changed vs Previous Flow
- Previous: wait for server → reload full list → cards appear
- Now: card appears instantly → server call in background → reconcile or rollback
- Removed `loadActiveList()` and `loadMyReview()` from success path (optimistic state already correct, only `loadCore()` for stats)
